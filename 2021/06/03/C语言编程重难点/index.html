C语言编程重难点1

发表于 2021-06-03

简介：暂无

stdin 输入

stdout 输出

fgets( num, sizeof(num) ,stdin ); //从stdin(标准输入-键盘)读取字符串到num数组中,最大可以读sizeof(num)-1个字符

puts fputs

const 修饰的局部变量不能通过变量本身修改值，可以用地址修改值，全局变量不能修改，因为属于常量区

野指针就是没有初始化的指针,指针的指向是随机的,不可以操作野指针

typedef：给类型起别名

1.简化结构体（struct）的关键字

void的使用

无类型，不可以创建变量，无法分配内存

限定函数返回值

限定函数中的参数列表

void * 万能指针

可以不需要强制类型转换 给其他指针赋值

用于接收未知指针类型，不通过强制类型转换就可以转为任意类型指针

sizeof 的使用

本质： 不是函数，而是一个操作符（运算符）

当统计类型占的内存空间时候，必须要加小括号

当统计变量占内存空间时候，可以不加小括号

返回值类型是 无符号整型 unsigned int

可以统计数组的长度（占用内存空间的大小）

数组名称如果在参数列表（作为形参）中，会退化为指针，指向数组的第一个元素，所以在函数外部和内部用sizeof计算数组长度不同

变量的修改方式

直接修改

间接修改 用指针去修改

通过指针对内存进行修改

用指针对自定义数据类型进行了修改

内存分区

运行前

代码区

​ 共享的

​ 只读的

数据区

​ data段 已初始化的全局变量、静态变量、常量

​ bss段(未初始化数据区) 未初始化的全局变量、静态变量、常量

运行后

栈区

​ 属于先进后出的数据结构

​ 由编译器管理数据开辟和释放

​ 变量的生命周期在该函数结束后自动释放掉

不要返回局部变量的地址，因为局部变量在函数执行之后就释放了，我们没有权限取操作释放后的内存

堆区

​ 容量远远大于栈

​ 没有先进后出这样的数据结构

​ 由程序员管理开辟和管理释放

​ malloc、free 在堆区开辟的数据，记住手动开辟的要手动释放

注意事项：如果在主调函数中没有给指针分配内存，那么被调函数中需要利用高级指针给主调函数中指针分配内存

​ 如果给主调函数中—个空指针分配内存,利用同级的指针是分配失败的。

​ 解决方式:利用高级指针修饰低级指计

数据区

​ 放入是静态变量、全局变量、常量

static 和 extern 区别

​ static 静态变量：编译阶段分配内存，只能在当前文件内使用，只初始化一次（内部链接属性）

​ extern 全局变量，C语言下默认的全局变量前都隐藏的加了该关键字（外部链接属性）

const修饰的变量

全局变量

​ 直接修改 失败

​ 间接修改 失败 原因放在常量区，受到保护

局部变量

​ 直接修改 失败

​ 间接修改 成功 原因放在栈上

​ 伪常量 不可以初始化数组

字符串常量 “hello”

​ ANSI没有指定出标准，不同的编译器可能有不同的处理方式

​ 有些编译器字符串常量可以共享

​ 有些编译器可以修改字符串常量，有些不可以

​ 有些编译器将相同的字符串常量看成同一个
